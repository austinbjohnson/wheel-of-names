## Notable Differences (from per-model write-ups)

- **Wheel rendering:** Claude 4.7 Opus used SVG slices; aqm20260522 used `<canvas>` slices. Both produced glassy per-slice gradients.
- **Pointer placement:** 3 o'clock (Claude 4.7 Opus) vs 12 o'clock (aqm20260522) — both satisfy "pointer clearly indicates winner."
- **Outcome model:** the key v4 requirement is "announced winner == slice under the pointer." aqm20260522 implements a *random-landing-then-evaluate* model (winner is read from where the wheel stops); the Claude 4.7 Opus build pre-aligns to a target with in-slice jitter. Worth scoring whether each is genuinely faithful.
- **Roster sourcing:** all models prefilled the Enterprise Zone PM list from available context (no web fetch). Roster accuracy varies by model and should be scored cautiously — names were mined from internal docs, not an authoritative directory.
