**Self-reported build details (from generation session):**
- Single-file HTML (no build step). Type: `Bricolage Grotesque` (display) + `Plus Jakarta Sans` (body) via Google Fonts — deliberately avoids Inter/Roboto/system stacks.
- Liquid glass treatment: `backdrop-filter: blur + saturate` on all panels; layered specular sheen over the wheel (dual radial-gradient highlight, `mix-blend-mode: screen`); animated orange/peach aurora blobs drifting behind the UI as the refractive light source; SVG fractal-noise grain overlay; inset highlight + rim shadows on the glass surfaces.
- Wheel: **canvas-rendered** slices (chosen over CSS-rotated DOM so labels stay crisp at any size and per-slice glass gradients composite cheaply), per-slice radial gradient for a glassy dome, glass center hub doubling as the Spin button, glass-droplet SVG pointer at **12 o'clock**.
- Spin logic: builds on running rotation (no snap-back); 6–8 full turns. **Honest outcome model** — the spin lands at a *random* angle, then a pure function `winnerAt(rotation)` reads whichever slice is physically under the pointer; the winner is an output of where it stops, not a pre-selected target. Tick sound on each slice crossing (WebAudio).
- Names list editable inline (click to edit), remove-on-`×`, add via form + Enter; Shuffle / Reset / Clear; persisted to `localStorage`.
- Confetti via Web Animations API; glass winner modal with "Remove from wheel / Spin again / Close".
